## README проекта

Основные желаемые разделы наполнения README.md проекта:

1. Должно быть простое описание для чего нужен проект, какая функциональность внутри реализована и как ее запустить + ссылка на раздел в вики (если такое есть)
1. Подготовка: где скачать неободимые веса, данные и тп. Куда их поместить, чтобы все работало или где указать к ним путь. Хорошо, если в скриптах проекта есть `export.sh` для подгрузки всего. Тут же про подготовку окружения (но вообще в проекте должен быть `make docker`, который собирает докер образ с готовым окружением).
1. Quick start (strongly recomended): как запустить код, НЕ МЕНЯЯ НИЧЕГО. То есть предполагает наличие каких-то bash файлов или сценариев в Makefile, реализующих демки трейна/инференса и тп, или хотя бы описание как это сделать питон скриптами БЫСТРО.
1. Краткое описание навигации внутри репы (как, где, что). Подробнее про используемые модели, какие параметры есть в обучении и инференсы, какие особенности. Тут уже подробнее стоит описать все процессы (как учить, как проверять, куда логировать и тп).

**⚠️⚠️ ATTENTION ⚠️⚠️**

- В данном шаблоне лишь примеры `Dokerfile`, `pyproject.toml` и `Makefile`. Эти файлики - **обязательные** атрибуты вашего проекта!
- STRONGLY RECOMMENDED: все зависимости и конфиги линтеров хранить в `pyproject.toml`, а быстрые команды в `Makefile`. Давайте не плодить 100500 файликов, я умоляю.
- В вашем проекте могут быть другие зависимости, доп сценарии и тп, это нормально. Также какие-то папки могут не пригодиться (например, `assets` или `notebooks`), тогда нет смысла их завозить.
- Poetry может не вывезти установку подходящего торча, тогда указываем в `pyproject.toml` в явном виде, откуда брать торч, например

```
torch = [
    { url = "https://download.pytorch.org/whl/cu118/torch-2.0.1%2Bcu118-cp39-cp39-linux_x86_64.whl", platform = "linux" },
    { url = "https://download.pytorch.org/whl/cpu/torch-2.0.1-cp39-none-macosx_11_0_arm64.whl", platform = "darwin" },
]
torchvision = [
    { url = "https://download.pytorch.org/whl/cu118/torchvision-0.15.2%2Bcu118-cp39-cp39-linux_x86_64.whl", platform = "linux" },
    { url = "https://download.pytorch.org/whl/cpu/torchvision-0.15.2-cp39-cp39-macosx_11_0_arm64.whl", platform = "darwin" },
]
```

- P.S. не забудь поменять имя проекта и автора в `pyproject.toml`

## Полезные команды:

- `make lint` - запуск линтеров (mypy, ruff, flake8, mdlint)
- `make refactor` - авторефакторинг (isort, black, mdformat, yamlfix, ruff)
- `make install` - установка всех зависимостей
- `make docker`, `make docker-push` - сборка и публикация Docker-образа
- `make dvc-init` - инициализация dvc (для версионирования данных/весов)
- `make test` - запусков тестов из папки "tests"
- `make smoke-test` - запусков fast-тестов из папки "tests" (необходимый минимум на очевидные ошибки)
